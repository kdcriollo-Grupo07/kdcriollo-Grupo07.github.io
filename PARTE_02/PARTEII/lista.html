Listas de nodos HTML DOM

NO es una matriz

  • Una lista de nodos puede parecer una matriz, pero no lo es. Puede recorrer una NodeList y consultar sus nodos por índice.
  • Pero no puede usar métodos de matriz como push(), pop() o join() en una lista de nodos. Una HTMLCollection es una colección de elementos de documento .
  • Una lista de nodos es una colección de nodos de documentos (nodos de elementos, nodos de atributos y nodos de texto). Se puede acceder a los elementos de HTMLCollection por su nombre, identificación o número de índice. Solo se puede acceder a los elementos de NodeList por su número de índice.
  • Los métodos getElementsByClassName() y getElementsByTagName() devuelven una HTMLCollection activa.
  • El querySelectorAll() método devuelve una lista de nodos estática.
  • La childNode spropiedad devuelve una lista de nodos activa.

Métodos que podemos utilizar

  • El objeto HTML DOM Nodelist: Un NodeList objeto es una lista (colección) de nodos extraídos de un documento.
    1. Un NodeList objeto es casi lo mismo que un HTMLCollection objeto.
    2. Algunos navegadores (más antiguos) devuelven un objeto NodeList en lugar de HTMLCollection para métodos como getElementsByClassName().
    3. Todos los navegadores devuelven un objeto NodeList para la propiedad childNodes.
    4. La mayoría de los navegadores devuelven un objeto NodeList para el método querySelectorAll().
  • Longitud de la lista de nodos HTML DOM: La length propiedad define el número de nodos en una lista de nodos
    La length propiedad es útil cuando desea recorrer los nodos en una lista de nodos
